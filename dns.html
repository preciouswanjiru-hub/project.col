Domain Name Server (DNS)

It is an internet service that translated human-readable names into machine-readable IP addressed. Operates in a hierachical and distributed manner. It is essential for the functioning of the internet, allowing users to access websites and other online resources using easy-to-remember domain names instead of complex IP addresses. DNS servers are responsible for resolving domain names to their corresponding IP addresses, enabling communication between devices on the internet.

Structure of DNS

    Root DNS Servers- The highest level of the DNS hierarchy, responsible for managing the top-level domains (TLDs) such as .com, .org and .net.
  1. Top-Level Domain (TLD) DNS Servers- These servers manage specific TLDs and are responsible for directing queries to the appropriate authoritative DNS servers for domains within their TLD.
  2. Second-Level Domains (SLD) DNS Servers- These servers manage domains that are one level below the TLD and are responsible for resolving queries for those specific domains.
  3. Authoritative DNS Servers- These servers hold the actual DNS records for specific domains and are responsible for providing the IP address associated with a domain name when queried.
DNS Resolution Process

When a user enters a domain name into a web browser, the DNS resolution process begins. The browser first checks its local cache for the IP address associated with the domain name. If it is not found, the browser sends a query to the configured DNS resolver, which then follows a series of steps to resolve the domain name to an IP address. The resolver may query multiple DNS servers, starting from the root DNS servers and working down through the hierarchy until it reaches the authoritative DNS server for the domain. Once the IP address is obtained, it is returned to the browser, allowing it to establish a connection with the web server hosting the requested resource.

Roles of Domain Name Servers

    Name Resolution- DNS servers are responsible for translating human-readable domain names into machine-readable IP addresses, allowing users to access websites and other online resources using easy-to-remember names.
  1. Load Distribution- DNS servers can distribute incoming traffic across multiple servers hosting the same website or service, helping to balance the load and improve performance.
  2. Caching- DNS servers can cache previously resolved domain names and their corresponding IP addresses, reducing the time it takes to resolve subsequent queries for the same domain and improving overall efficiency.
  3. Security- DNS servers can implement security measures to protect against various types of cyber threats, such as DNS spoofing and DDoS attacks. They can also support secure communication protocols like DNSSEC to ensure the integrity and authenticity of DNS responses.
